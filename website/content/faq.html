FAQ

  1. Having performance issues?

    See your doctor--ED is treatable.
    OR
    Try using namespaces and shorten resource IDs. However, performance is usually related to memory constraints. Modify the OpenRefine memory allocation parameters in its refine.ini. The variable "REFINE_MEMORY" controls the maximum memory allocation pool. The variable "REFINE_MIN_MEMORY" controls the initial memory allocation. These variables correspond to the JVM -Xmx and -Xms parameters. The defaults may be too small for your use.
    Additionally, consider slicing your data file into smaller chunks. An RDF Transform can be applied to multiple files of the same structure:

    1. Export the RDF Transform from the first file's work.
    2. Importing the RDF Transform for the successive files.

  2. A SPARQL endpoint is not responding the way I want!
    A SPARQL endpoint doesn't have text searching!

    Try using a DESCRIBE query on a known resource.

  3. How do I get a reconciled IRI in the RDF exporter?

    Use OpenRefine's Reconciliation functions. Then, use the GREL expression cell.recon.match.id to automagically match each cell value to a reconciled resource.
